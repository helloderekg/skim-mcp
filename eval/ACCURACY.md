*still answer correctly, and what does the answer actually cost?* This table prices the

These fixtures are deliberately SMALL (a 46-line module, a short log) - near skim's
worst case, since the skeleton wrapper is a fixed cost. On large files the same
mechanism measures 30-91% per task (median 51% across 24 stdlib modules; see
[BENCHMARKS.md](../BENCHMARKS.md)). Costs exclude the JSON tool-result envelope,
which both approaches pay.

## The correctness half (live protocol)

Oracle cost assumes the model expands exactly what it needs. Whether it DOES is the
under-fetch question, and it is measured, not assumed:

1. Mount skim and open a **fresh** session (no hints beyond 'use skim').
2. Ask the seven questions in [QUESTIONS.md](QUESTIONS.md) against the fixtures.
3. `uv run python eval/score_expand_loop.py` scores the call log mechanically:
   expand-when-needed rate (pass bar >= 80%), confident-wrong-without-expand (< 10%),
   over-expansion (watch).

## Scoring another tool on the same yardstick

Report, per question: **(1)** total tokens shown to the model to reach the answer, and
**(2)** whether the answer-bearing lines were available to the model *verbatim* at answer
time (a lossy repo-map or paraphrasing summarizer scores no on 2 - if the surviving text
happens to contain the answer, say so, but paraphrase is not verbatim). PRs adding runner
adapters for other tools are welcome; the fixtures and markers are in this file's `TASKS`.
